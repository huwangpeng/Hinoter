# 两份 `.hinote` 样本解析结果

## 结论

两份文件都没有整体加密，是普通 ZIP 包。`.jhinote` 数据可无损解压成 JSON；原笔记的可见内容由图片资源和私有笔迹二进制共同组成。

- 页面元素：一张 `2800 x 1222` 的 JPEG 题图
- 手写层：`df0e8da9dcdbaec42f0b8e0ce134b345.bin`，文件头 `PENCILENGINE`

### 无界笔记

- 标题：未命名笔记 (5)
- 笔记 ID：`34fe511b84774cbdaa21ce8a12648821`
- 页面元数据：4 条，页码为 1、2、3、6
- 当前视图：导航记录有 3 个画布位置，当前索引为 2

## 结构差异

| 项目 | 有界笔记 | 无界笔记 |
| --- | --- | --- |
| 容器 | ZIP | ZIP |
| 元数据 | GZIP JSON | GZIP JSON |
| 画布 | 固定页面 | 无限画布分块 |
| 笔迹格式 | `PENCILENGINE` | `PENKITINFENG` |
| 可读资源 | JPEG 题图、PNG 缩略图 | PNG 当前视图缩略图 |
| 页面元素 | 1 个图片元素 | 无，内容在笔迹块中 |

工具会完整导出 JSON、资源和私有二进制，并计算 SHA-256，避免解析过程改变原始证据。

## 矢量导出修订

有界笔记导出器现会读取 `PENCILENGINE` 点表中的笔刷基准宽度和实际压感字段。当前样本恢复出 32 个笔画，压感范围为 `0.2` 至 `0.880172`，导出的 SVG/PDF 含 13 种线宽，范围为 `8.0` 至 `26.405`。固定页 PDF/SVG 同时恢复 `pageColor` 背景和图片元素的位置、尺寸、角度；JPEG 以原始 DCT 数据嵌入 PDF，不经重采样。

无边 `PENKITINFENG` 的笔画分块格式仍需单独解码，尚不能保证正确渲染时，工具不会输出不可信的伪矢量结果。

## 大样本格式复核

`1.hinote` 含 25 个有界页面，每页都有 `PENCILENGINE` 笔迹数据和预览图。重新以该样本反查后，确认主笔迹的记录链为“60 字节样式记录、16 字节点表头、N 个 36 字节点、64 字节索引记录”。旧版扫描式解析会把索引和辅助几何误当笔画，因此会造成文字扭曲；当前导出器只使用这条主记录链。
